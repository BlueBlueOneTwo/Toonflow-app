版本：v1.0 | 日期：2026-03-30 | 状态：设计草稿

## 一、设计理念

### 核心关键词

**科技感 · 沉浸式 · 极简主义 · AI Native**

剧灵不是传统工具类产品，而是一个 AI 驱动的创作平台。界面应该传达"智能正在为你工作"的感觉——不是冰冷的表单，而是一个有温度、有节奏、有反馈的创作空间。

### 情感定位

| 维度 | 感受 | 视觉语言 |
|------|------|---------|
| 智能感 | AI 在认真工作 | 流动的数据、渐变的粒子、光效 |
| 安全感 | 一切可控 | 清晰的进度、明确的按钮、状态可见 |
| 创作感 | 正在创作好作品 | 大面积画布、沉浸式工作区、零干扰 |
| 高端感 | 专业级工具 | 精致细节、克制用色、精准动效 |

### 2.3 动效原则

**核心原则**：动效服务于反馈，不是装饰。

| 类型 | 场景 | 时长 | 曲线 |
|------|------|------|------|
| **微交互** | 按钮 hover、icon 变化 | 150ms | ease-out |
| **状态切换** | 展开/收起、切换面板 | 250ms | ease-in-out |
| **加载进度** | 任务生成中、文件上传 | 持续 | linear + 脉冲 |
| **页面过渡** | 路由切换、弹窗出现 | 300ms | ease-out |
| **数据流动** | 进度条、数字跳动 | 600ms | spring |

**AI 工作状态动效**：
- 生成中：呼吸式光晕 + 粒子流动（渐变色光点沿路径移动）
- 思考中：脉冲圆环（类似雷达扫描效果）
- 完成：涟漪扩散 + 短暂闪光


## 四、关键页面设计

### 4.1 工作台（Home）

**布局**：左侧项目列表 + 中间快捷生成 + 右侧最近动态

**AI 生成入口**（核心 CTA）：
- 大面积渐变卡片："告诉我你的故事，AI 帮你成片"
- 输入框支持：粘贴文字 / 上传文件 / 输入大纲
- 风格选择（横向滑动卡片）：古风 / 甜宠 / 霸总 / Vlog风 / 漫改风…
- 点击后进入创作画布

**设计细节**：
- 项目卡片：封面图 + 标题 + 状态标签 + 操作菜单
- 状态徽章：生成中（紫色脉冲）/ 已完成（绿色）/ 草稿（灰色）
- 右下角悬浮：快速新建按钮（圆形，渐变，hover 展开 3 个选项）

---

### 4.2 创作画布（核心创作区）

**三栏布局**：
```
左栏（可折叠）：        中间（主画布）：          右栏（可收起）：
大纲树 / 章节          剧本预览 / 分镜           属性面板：
- 章节列表              分镜缩略图               - 角色设定卡
- 跳转导航              预览图                   - 风格配置
- 快捷操作              生成状态                  - AI 建议
```

**底部工具栏**：
```
[AI 剧本] [AI 分镜] [AI 图片] [AI 视频]  │  [预览] [下载] [分享]
```

**AI 生成状态层**（覆盖在整个画布上方）：
- 生成中：半透明遮罩 + 粒子流动动画 + 步骤说明（"正在生成第 3/10 个分镜…"）
- 完成后：涟漪动画 + 通知提示

---

### 4.3 登录/注册页

**设计要求**：
- 全屏渐变背景（深空 + 粒子动效）
- 居中卡片（毛玻璃效果）
- 微信扫码为主（登录按钮上方显眼位置）
- 邮箱登录为辅（Tab 切换）
- 游客体验入口：底部小字"先体验一下 →"（跳转后限制额度）

---

### 4.4 订阅中心

**布局**：套餐卡片 + 积分包 + 历史订单

**设计细节**：
- 套餐对比表（横向排布当前推荐高亮）
- 每个套餐卡片包含：价格 / 额度 / 主要权益列表 / CTA 按钮
- 免费档：灰色调 + "立即体验"
- 创作档：主推款（渐变边框高亮）
- 专业档：深色高端风格
- 企业档：最小卡片（"联系商务"）

## 五、AI 生成状态设计（重点）

这是科技感最重要的体现。

### 5.1 全流程生成状态

**Phase 1：剧本生成**
```
状态：🌟 智能分析中
动效：光点从文本框向中央汇聚，汇聚后形成结构化卡片
进度：0-100%（文字百分比跳动）
完成：卡片翻转动画，展开剧本内容
```

**Phase 2：分镜生成**
```
状态：🎬 分镜设计中
动效：屏幕边缘出现分格线动画（模拟电影分镜感）
逐个生成：每个分镜缩略图从网格中弹出
完成：网格补全 + 整体微光闪烁
```

**Phase 3：视觉生成**
```
状态：✨ 渲染中
动效：每个分镜独立加载进度（圆形进度环）
背景：流动的粒子网格（AI 正在工作感）
完成：图片从模糊到清晰（3D 聚焦效果）
```

**Phase 4：视频合成**
```
状态：🎥 合成中
动效：分镜片段依次滑入时间轴
底部：音频波形可视化
完成：播放按钮出现（发光效果）
```

**Phase 5：成片完成**
```
状态：✅ 完成
动效：全屏淡入 → 作品卡片弹出
通知：右上角 + WebSocket 推送 + 浏览器通知
```

### 5.2 错误状态设计

- API 超时：卡片抖动 + 橙色提示 + "重试"按钮
- 质检未通过：红色边框 + 具体问题标注 + "查看原因"
- 积分不足：弹出充值引导（不打断主流程）

## 六、响应式策略

| 断点 | 布局调整 |
|------|---------|
| ≥1440px | 三栏布局（侧边+画布+属性） |
| 1024-1439px | 侧边收起为 icon，属性栏叠加 |
| 768-1023px | 底部 Tab 导航，侧边变抽屉 |
| <768px | 全屏单页模式，底部工具栏 |

**移动端 H5**：
- 简化顶部导航（Logo + 用户头像）
- 主要操作收到底部固定区
- 长列表改用卡片轮播
- AI 生成进度：独立全屏页面（防止切换丢失）

## 八、技术实现要点

### 8.1 状态管理

```typescript
// 使用 Pinia（现有）

stores/
├── useAuthStore.ts       // 登录状态 + 用户信息
├── useSubscriptionStore.ts  // 套餐 + 积分
├── useProjectStore.ts    // 当前项目 + 上下文
├── useWorkflowStore.ts   // 生成进度 + 实时状态
└── useUIStore.ts          // 侧边栏折叠 / 主题偏好
```

### 8.2 实时通信

```typescript
// WebSocket（复用现有 express-ws）

// 连接：登录后自动建立
// 消息类型：
// - task:progress  → 生成进度更新（0-100）
// - task:complete → 任务完成通知
// - task:failed   → 失败 + 错误原因
// - ai:suggestion → AI 主动建议（当前分镜可优化）
```

### 8.3 性能优化

- **路由懒加载**：所有页面组件使用 `defineAsyncComponent`
- **图片**：WebP 格式 + 懒加载 + LQIP 占位
- **大数据**：虚拟列表（剧本/分镜列表）
- **首屏**：骨架屏 + 渐进加载
- **构建**：Vite 代码分割 + 分析

## 九、设计检查清单

| 检查项 | 标准 |
|--------|------|
| 颜色一致性 | 所有色值来自 Design Tokens，禁止硬编码 |
| 动效一致性 | 统一动效参数（时长/曲线），禁止随机值 |
| 响应式 | 三个断点（1440/1024/768）完整测试 |
| 可访问性 | WCAG AA 标准（对比度/焦点/替代文本）|
| 加载状态 | 每个异步操作必须有 loading 或 skeleton |
| 错误状态 | 每个错误路径有明确提示 + 可操作按钮 |
| 空状态 | 每个列表/画布有空白状态设计（不是一片白）|
