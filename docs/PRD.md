> 状态：草稿 | 创建时间：2026-03-30 | 负责人：Bruce + 小咕嘟

---

## 一、产品愿景

**一句话定位**：
把好的故事，变成让人上头的短剧——不需要AI知识，不需要剪辑技能，只要有个好故事，AI帮你从头做到尾。

**核心价值主张**：
我们提供稳定可靠的 AI 能力组合和经过打磨的工作流，客户只需提供创意和内容，AI 输出可发布的短剧成片。

---

## 二、目标用户

| 用户画像 | 特征 | 核心需求 |
|---------|------|---------|
| 个人创作者 | 小说作者、兼职短视频博主 | 快速把文字变成视频，不懂技术 |
| 小型工作室 | 1-3人短剧团队 | 提效率、降成本、保证出品稳定 |
| MCN机构 | 有批量生产需求 | 批量任务、团队协作、品牌风格统一 |
| 技术爱好者 | AI/开发者用户 | 定制模型、自定义工作流 |

## 三、核心功能需求

### 3.1 用户端功能（按优先级）

#### P0 - MVP 必须有

**1. 用户注册与认证**
- 邮箱/手机号注册，密码登录
- 第三方登录（微信/Google）
- 游客体验入口（限制额度）

**2. 订阅与积分系统**
- 基础订阅：月度套餐，按月收费
- 积分充值：超额生成时扣除
- 计费单位：以"集"为单位（1集 ≈ 1-3分钟短剧）
- 套餐示例：
  - 免费档：每月 2 集体验额度，带水印
  - 创作档 ¥99/月：每月 5 集额度，无水印
  - 专业档 ¥299/月：每月 15 集额度 + 批量任务 + 高清输出
  - 企业档：定制报价，API 接入，团队协作

**3. 故事输入**
- 支持上传 TXT/Word 小说文件
- 支持直接粘贴文字
- 支持故事大纲输入（AI 扩展为完整剧本）
- 章节拆分与选择

**4. AI 全自动工作流（核心 Pipeline）**

```
用户故事/小说
    ↓
【剧本生成】Gemini 文本理解 → 结构化剧本（对白、场景、情绪标注）
    ↓
【角色管理】AI 识别生成角色设定（外貌、性格、身份），支持手动编辑
    ↓
【分镜制作】剧本 → 分镜提示词（前景/中景/背景、角色动态、道具、场景布局）
    ↓
【视觉生成】Seedance 2.0 / 豆包 → 分镜图 → 视频片段
    ↓
【后处理】FFmpeg → 片段拼接 + 字幕压制 + 音频合成 + 调色
    ↓
【质检节点】每个环节输出检测，不合格打回重写
    ↓
可发布成片（MP4）
```

**5. 模型路由层（技术底座）**
- 定义标准接口协议，支持多模型热插拔
- 当前主力组合：Gemini（文本）+ Seedance 2.0/豆包（视频）+ FFmpeg（剪辑）
- 配置文件驱动，方便切换/新增模型

**6. 作品管理**
- 生成记录列表
- 在线预览 / 下载 MP4
- 作品分享（带水印展示页）

---

#### P1 - 差异化功能

**7. 风格模板系统**
- 预置风格：古风、甜宠、霸总、复仇、悬疑、都市
- 每个风格绑定对应提示词工程，一键切换
- 用户可保存自定义模板

**8. 角色一致性管理**
- 生成的图片中角色外观保持一致
- 支持换装/换场景时保持角色特征
- 多集连续剧模式下角色记忆

**9. 质检与反馈**
- 每环节输出质量评分
- 不合格自动重写（最多3次）
- 用户可手动调整各环节输出

**10. 任务队列与批量生成**
- 支持一次排队生成多集
- 后台任务管理，进度实时监控
- 中断恢复（任务状态持久化）

---

#### P2 - 运营功能

**11. 作品展示厅**
- 优秀作品公开展示（带水印）
- 用户可分享到外部平台
- 形成口碑传播

**12. 自定义模型接口**
- 技术用户可接入自己的模型
- 标准协议格式：`<<CUSTOM:model_name>>` 触发
- 沙盒隔离，平台不介入计费

**13. 模板/剧本市场**
- 用户可分享自己调好的模板
- 其他用户可付费/积分使用
- 平台抽佣（可选，后期）

---

#### P3 - 长期规划

**14. 多集连续剧模式**
- 用户输入故事大纲，AI 生成多集连贯剧情
- 集与集之间剧情衔接一致

**15. 旁白配音**
- 文字转语音（TTS）
- 多音色选择

**16. API 开放**
- 对企业客户提供 API 接入

---

- 套餐定价管理

## 四、非功能需求

### 4.1 技术要求

- **多租户隔离**：每个用户数据完全隔离
- **高可用**：核心 pipeline 支持水平扩展
- **稳定性**：80%+ 良品率（通过质检节点保证）
- **响应速度**：用户可感知进度（WebSocket 实时推送）
- **存储**：生成的视频文件支持 OSS/S3 存储

### 4.2 合规要求

- 用户上传内容需原创声明
- 提供"原创故事编辑器"降低版权风险
- 成品默认带平台水印（免费档），付费用户可去除
- 内容安全审核机制（文本 + 视频）

## 五、商业模式

### 5.1 收入结构

| 收入项 | 定价参考 |
|--------|---------|
| 基础订阅（月费） | ¥99 - ¥299/月 |
| 超额积分充值 | ¥9.9/100积分 ≈ ¥0.099/集 |
| 无水印输出 | 免费/付费分水岭 |
| 高清 1080P+ | +50% 积分 |
| 角色一致性增强 | +30% 积分 |
| 配乐/BGM | +20% 积分 |
| 旁白配音 | 按字数计积分 |
| 企业定制 | 单独报价 |

### 5.2 成本结构（供参考）

- LLM API 调用成本（Gemini）
- 视频生成 API 成本（Seedance 2.0 / 豆包）
- 存储成本（OSS）
- 服务器成本

**定价原则**：视频生成成本约占 60-70%，LLM 约占 10-15%，其余为运营/利润。

## 六、技术架构（概述）

### 6.1 系统架构

```
客户端（Web / 移动端 H5）
    ↓ HTTPS
Nginx 负载均衡
    ↓
API 网关（认证 + 路由）
    ↓
业务层
├─ 用户服务（注册/订阅/积分）
├─ 工作流引擎（剧本→分镜→视频→剪辑）
├─ 模型路由层（Gemini / 豆包 / Seedance / FFmpeg）
├─ 质检节点（各环节输出验证）
├─ 任务队列（异步任务 + WebSocket 推送）
└─ 存储服务（文件上传 / 视频输出）
    ↓
数据库（PostgreSQL）
对象存储（OSS/S3）
缓存（Redis）
```

### 6.2 技术选型建议

| 模块 | 建议方案 |
|------|---------|
| 后端框架 | 沿用现有 Node.js + TypeScript |
| 数据库 | PostgreSQL（用户、订单、任务数据） |
| 对象存储 | 阿里云 OSS 或 AWS S3 |
| 任务队列 | BullMQ（基于 Redis）|
| 实时推送 | WebSocket（任务进度）|
| 前端 | 沿用 Vue 3 + TypeScript（Toonflow-web）|
| 视频处理 | FFmpeg（本地或云函数）|
| AI 模型路由 | 独立模块，配置文件驱动 |

### 6.3 模型路由协议（草案）

每个模型需实现标准接口：

```typescript
interface AIModel {
  type: 'llm' | 'image' | 'video' | 'audio';
  name: string;
  endpoint: string;
  apiKey: string;
  capabilities: string[];
  costPerCall: number;
  call(input: ModelInput): Promise<ModelOutput>;
}
```

## 七、开发优先级（推荐）

### 第一阶段：MVP（6-8周）
1. 用户注册/登录 + 订阅系统
2. 故事输入 → 剧本生成（Gemini）
3. 分镜生成
4. 视频生成（Seedance / 豆包）
5. FFmpeg 整合（字幕 + 拼接 + 音频）
6. 作品预览 / 下载
7. 质检节点（基础版）

### 第二阶段：差异化（4-6周）
1. 风格模板系统
2. 角色一致性管理
3. 任务队列 + 批量生成
4. WebSocket 进度推送

### 第三阶段：运营增强（4-6周）
1. 作品展示厅
2. 自定义模型接口
3. 模板市场
4. 数据分析面板

## 八、风险与挑战

| 风险 | 应对策略 |
|------|---------|
| 视频 API 不稳定/涨价 | 设计降级方案，保留2个以上备选 |
| 生成质量不稳定 | 质检节点 + 人工抽检 |
| 版权争议 | 用户原创声明 + 合规审核 |
| 竞品复制 | 工作流打磨是护城河，快速迭代 |
| 冷启动难 | 先在垂直社群验证（短剧创作者群）|
