## 一、产品愿景

**一句话定位**：
把好的故事，变成让人上头的短剧——不需要AI知识，不需要剪辑技能，只要有个好故事，AI帮你从头做到尾。

**核心价值主张**：
我们提供稳定可靠的 AI 能力组合和经过打磨的工作流，客户只需提供创意和内容，AI 输出可发布的短剧成片。

**解决的核心问题**：
短剧创作者（小说作者、短视频博主、小型工作室）有内容创意，但缺乏视频制作能力——不懂AI、不会剪辑、无法承担外包费用。传统制作一集短剧需要 3-5 人团队 + 数千元成本，我们让一个人 + 一个平台就能完成。

## 二、目标用户与场景

### 用户画像 1：个人创作者（小说作者 / 兼职博主）

**特征**：有文字内容创作能力（写小说、写故事），想快速把文字变成视频发布到抖音/快手，但完全不懂 AI 和视频制作。

**使用场景**：
> 李梅是一位写穿越小说的作者，在晋江有 5000 粉丝。她在飞书看到 Toonflow 的推广，注册后上传了自己的小说第一章（共 3000 字），选择"古风甜宠"风格模板，点击"生成"。15 分钟后，她收到通知——完整的一集短剧（90 秒，带字幕和 BGM）已经生成好了，她直接下载 MP4 发布到抖音。

**核心需求**：操作极简、效果够用、发布时间快

---

### 用户画像 2：小型工作室（1-3 人短剧团队）

**特征**：已经有短剧账号，内容需求大，但人工制作成本高、效率低。希望用 AI 提效。

**使用场景**：
> 张老板运营一个短剧账号，每周更新 3 集。用传统方式需要：编剧（3000 元/集）+ 拍摄（5000 元/集）+ 剪辑（2000 元/集）= 1 万元/集。使用 Toonflow 后，同等质量成本降至约 500 元/集（含 API 成本），团队从 5 人缩减到 1 人（负责审核和发布），每周更新能力从 3 集提升到 10 集。

**核心需求**：批量生产、质量稳定、成本可控

---

### 用户画像 3：MCN 机构（有批量需求的内容团队）

**特征**：需要管理多个账号、多个人员协同，要求输出风格统一、品牌一致。

**使用场景**：
> MCN 机构旗下有 20 个短剧账号，每个账号有不同的内容定位（如霸总、甜宠、悬疑）。Toonflow 的风格模板系统让每个账号绑定自己的模板，批量生成时角色风格一致，多人协作时项目相互隔离，互不干扰。运营人员可以在一个后台管理所有账号的生成任务。

**核心需求**：多账号管理、团队协作、品牌风格统一

---

### 用户画像 4：技术爱好者（AI/开发者）

**特征**：有技术背景，想接入自己的模型、自定义工作流，有极客精神。

**使用场景**：
> 程序员小王发现平台的内置模型在生成古装场景时效果一般，他自己有一个专门针对古风训练的模型。他通过平台的 `<<CUSTOM:my-ancient-style-v1>>` 接口注册了自己的模型，配置触发提示词后，在生成古装类内容时自动调用他的模型，输出效果显著提升，且 API 费用由他自己承担。

**核心需求**：自定义模型接入、灵活工作流配置

## 三、功能需求详述（SMART 原则）

每个需求格式：用户 + 操作 + 结果 + 解决问题

---

### P0 - MVP 必须有

#### F0-1：用户注册与认证

**场景**：新用户第一次使用

**用户操作**：
- 输入邮箱和密码注册账号
- 或使用微信一键登录（扫码授权）
- 登录后进入工作台

**系统行为**：
- 注册时发送邮箱验证链接，30 分钟内有效
- 微信登录首次需绑定邮箱（用于账号安全）
- 登录成功后显示剩余集数额度

**解决的实际问题**：账号安全 + 套餐隔离

---

#### F0-2：订阅与积分系统

**场景**：用户想知道自己能生成多少集，以及如何充值

**套餐详情**：

| 套餐 | 价格 | 每月额度 | 超出积分 | 水印 | 输出质量 |
|------|------|---------|---------|------|---------|
| 免费档 | ¥0 | 2 集/月 | 无 | 带平台水印 | 720P |
| 创作档 | ¥99/月 | 5 集/月 | ¥9.9/100积分 | 无 | 1080P |
| 专业档 | ¥299/月 | 15 集/月 | ¥8/100积分 | 无 | 1080P |
| 企业档 | 定制 | 定制 | 定制 | 无 | 4K |

**用户操作**：
- 在套餐页面选择套餐 → 点击购买 → 微信/支付宝支付 → 自动开通
- 积分不足时提示充值，支持单独购买积分包（100/500/1000 积分）
- 查看"积分明细"了解每笔消耗（生成哪集、扣了多少积分）

**解决的实际问题**：按需付费，降低试用门槛

---

#### F0-3：故事输入与预处理

**场景**：用户把小说内容导入系统

**用户操作**：
1. **上传文件**：支持拖拽 TXT / .docx 文件，文件大小上限 5MB
2. **粘贴文字**：直接将小说文字粘贴到文本框
3. **输入大纲**：提供一个故事梗概（100-500 字），AI 扩展为完整剧本

**系统行为**：
- 文件上传后，自动解析章节结构（识别"第一章"、"Chapter 1"等模式）
- 展示章节列表，用户可勾选要生成的章节（全选/反选）
- 对于粘贴文字或大纲输入，系统提示输入字数并预估可生成的集数
- 检测内容是否包含敏感词，如有则拦截并提示修改

**解决的实际问题**：用户不需要懂格式，系统自动处理文本结构

---

#### F0-4：AI 全自动工作流

**场景**：用户提交故事后，系统自动完成从文字到视频的全流程

**Step 1 - 剧本生成（Gemini）**
- 用户操作：点击"开始生成"，选择风格模板（可选）
- 系统行为：
  - 将小说文本发送给 Gemini，分析文本结构（场景、对白、人物动作、情绪）
  - 输出结构化剧本：包含场次编号、场景描述、角色对白、情绪标签、镜头建议
  - 生成后展示剧本预览，用户可手动编辑任意段落
  - 确认剧本后点击"下一步"

**Step 2 - 角色管理**
- 系统行为：
  - AI 自动识别剧本中的角色，生成角色设定卡（外貌描述、性格标签、身份背景）
  - 同步生成每个角色的参考图片（用于后续图片/视频生成的一致性）
- 用户操作：
  - 查看角色设定卡，可修改角色描述
  - 可上传角色参考图（替代 AI 生成的图片）
  - 可增加/删除角色
- 解决的实际问题：角色一致性是短剧质量的核心，用户必须能管理角色

**Step 3 - 分镜制作**
- 系统行为：
  - 根据剧本自动生成分镜列表，每个分镜包含：
    - 分镜编号
    - 场景描述（室内/室外、白天/夜晚）
    - 画面构图（前/中/后景）
    - 角色动作
    - 镜头类型（特写/全景/推镜/拉镜）
  - 为每个分镜生成 AI 提示词（用于后续图片生成）
- 用户操作：
  - 查看分镜列表，可调整分镜顺序、合并或拆分分镜
  - 可手动编辑每个分镜的描述和提示词
  - 确认后点击"生成视频"

**Step 4 - 视觉生成（Seedance 2.0 / 豆包）**
- 系统行为：
  - 按顺序调用视频生成 API，将分镜转化为视频片段
  - 每个分镜生成 1 个视频片段（5-15 秒）
  - 生成过程中实时显示进度（"正在生成第 3/10 个分镜…"）
- 用户操作：
  - 可在生成过程中暂停/取消
  - 完成后可对单个片段重新生成（点击片段旁的"🔄"）
  - 可手动上传替换某个片段

**Step 5 - 后处理（FFmpeg）**
- 系统行为：
  - 将所有视频片段按顺序拼接为一个完整视频
  - 自动生成字幕（语音转文字 + 时间轴对齐）
  - 压制字幕到视频（可选择字幕样式：颜色、位置、字体大小）
  - 自动添加 BGM 背景音乐（从预设曲库中选择，或用户上传）
  - 最终输出 MP4 文件，分辨率 720P/1080P 可选

**解决的实际问题**：用户只需点"开始"，其余全部自动完成

---

#### F0-5：质检与质量控制

**质检节点设计**（每环节自动检测）：

| 环节 | 质检内容 | 不合格处理 |
|------|---------|-----------|
| 剧本 | 剧情逻辑漏洞、角色行为矛盾 | 自动重写（最多3次），仍不合格则提示用户手动调整 |
| 分镜 | 分镜与剧本一致性、描述完整性 | 提示具体问题所在，用户可一键修复 |
| 视频片段 | 图片清晰度、构图合理性 | 自动重新生成该片段 |
| 最终成片 | 音画同步、字幕对齐、视频完整 | 整体质量评分（0-100），低于 60 分提示重新生成 |

**用户操作**：
- 系统中显示每个环节的质检结果（通过/不通过）
- 不合格项高亮显示，用户可查看原因并手动调整
- 用户可选择"接受当前版本"跳过质检继续下一步

**解决的实际问题**：保证 80%+ 良品率，减少用户反复修改的时间

---

#### F0-6：作品管理

**用户操作**：
- 在"作品管理"页面查看所有生成记录
- 每个作品显示：生成时间、使用的模板、时长、分辨率、状态
- 点击作品可预览、下载（MP4）、重新生成、删除
- 支持分享：生成带水印的分享链接，可一键复制到抖音/快手发布

**解决的实际问题**：生成后可管理、可发布、可追溯

---

### P1 - 差异化功能

#### F1-1：风格模板系统

**用户操作**：
- 在生成流程中选择风格模板（古风/甜宠/霸总/复仇/悬疑/都市等）
- 每个风格包含：画面风格描述、角色服装规范、镜头语言偏好、BGM 风格
- 模板预览展示该风格的参考图
- 用户可保存自己调整过的模板（命名为"我的古风模板"）

**解决的实际问题**：不同内容类型需要不同风格，一键切换降低学习成本

---

#### F1-2：角色一致性管理

**用户操作**：
- 在角色管理页面，每个角色显示"一致性分数"（AI 评估）
- 支持上传角色多角度参考图（正面、侧面、表情）
- 多集生成时，角色记忆功能自动保持角色外观一致

**解决的实际问题**：短剧多集之间同一角色外观差异大是最常见的质量问题

---

#### F1-3：任务队列与批量生成

**用户操作**：
- 选择多集同时生成（最多 10 集），进入批量队列
- 后台任务页面显示队列状态（排队中/生成中/完成/失败）
- 实时进度推送（浏览器通知或站内消息）
- 支持中断恢复（服务器重启后自动继续未完成的任务）

**解决的实际问题**：批量生产需求（MCN 机构）和长时间生成的可靠性

---

### P2 - 运营功能

#### F2-1：作品展示厅

**用户操作**：
- 生成的优秀作品（用户授权）出现在展示厅
- 其他用户可观看、点赞、评论
- 展示厅作品带平台水印，不可直接下载

**解决的实际问题**：社区氛围 + 口碑传播 + 展示产品能力

---

#### F2-2：自定义模型接口

**用户操作**：
- 在"开发者设置"中注册自定义模型（提供 endpoint + API key + 能力描述）
- 配置触发条件：当用户输入包含 `<<CUSTOM:model_name>>` 时调用该模型
- 自定义模型产生的 API 费用由用户自行承担，平台不介入

**解决的实际问题**：满足技术用户的深度定制需求

---

### P3 - 长期功能

#### F3-1：多集连续剧模式

**用户操作**：
- 输入一个完整的故事大纲（500-2000 字）
- AI 自动拆分为多集（3-10 集可选）
- 每集剧情衔接一致，角色状态跨集记忆
- 批量生成多集，最后统一后处理

**解决的实际问题**：做系列短剧的创作者（这是高价值场景）

---

#### F3-2：旁白配音

**用户操作**：
- 在后处理阶段选择是否添加旁白配音
- 选择音色（温暖男声/知性女声/低沉男声等）
- 旁白文字自动从剧本中提取，用户可修改

**解决的实际问题**：有配音的短剧完播率显著高于无声视频

## 四、非功能需求

### 4.1 技术要求

- **多租户隔离**：每个用户的数据（作品、角色、积分）完全隔离，数据库层面按 user_id 分区
- **高可用**：API 服务支持多实例部署，核心生成 pipeline 支持水平扩展
- **稳定性**：80%+ 良品率（通过质检节点 + 自动重写机制保证）
- **响应速度**：生成过程实时推送（WebSocket），用户每 10 秒更新一次进度
- **存储**：视频文件存储在 OSS/S3，用户历史作品保存 30 天（免费档）/永久（付费档）
- **API 限流**：防止单用户过度消耗资源，限制每分钟调用次数

### 4.2 合规要求

- 用户上传内容需勾选"原创声明"（上传即代表原创或已获授权改编）
- 提供"原创故事编辑器"，降低版权风险
- 成品默认带平台水印（免费档），付费用户可去除
- 内容安全审核：文本层面（敏感词过滤）+ 视频层面（截图抽帧检测）
- 符合国内短视频平台（抖音/快手）的发布规范

|------|------|------|
| 免费档 | ¥0 | 2 集/月，带水印，720P |

- **存储（OSS）**：约占 5%（视频文件存储）
- **服务器 + 带宽**：约占 10-15%
- **运营 + 利润**：约 5-10%

## 六、技术架构（概述）

### 系统架构

```
用户端（Web / H5）
    ↓ HTTPS
Nginx（负载均衡）
    ↓
API 网关（认证 + 限流）
    ↓
业务层
├── 用户服务（注册/登录/订阅/积分）
├── 工作流引擎（剧本→分镜→视频→剪辑）
├── 模型路由层（Gemini / Seedance / 豆包 / FFmpeg / 自定义模型）
├── 质检节点（每环节输出验证）
├── 任务队列（BullMQ + Redis，异步处理）
└── 存储服务（文件上传 / 视频输出）
    ↓
PostgreSQL（用户、订单、任务数据）
Redis（缓存、任务队列、实时状态）
OSS/S3（视频文件、生成资产）
```

### AI 模型路由协议

每个模型实现标准接口：

```typescript
interface AIModel {
  type: 'llm' | 'image' | 'video' | 'audio' | 'tts';
  name: string;
  provider: string;
  endpoint: string;
  apiKey: string;
  capabilities: string[];  // 如 ['script-generation', 'character-analysis']
  costPerCall: number;      // 单次调用成本（积分）
}

// 模型注册示例
{
  type: 'llm',
  name: 'gemini-2.0-flash',
  provider: 'google',
  capabilities: ['script-generation', 'character-analysis', 'scene-description'],
  costPerCall: 5
}
```

## 八、风险与挑战

| 风险 | 应对策略 |
|------|---------|
| 视频 API 不稳定/涨价 | 设计降级方案，保留 2 个以上备选服务商 |
| 生成质量不稳定 | 质检节点 + 自动重写 + 人工抽检机制 |
| 版权争议 | 用户原创声明 + 合规审核 + 原创编辑器 |
| 竞品复制 | 工作流打磨是护城河，快速迭代 |
| 冷启动难 | 先在垂直社群验证（短剧创作者群、小红书小说作者群）|
